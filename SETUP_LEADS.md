# Подключение заявок: всё падает в Google Таблицу

Каждая заявка с квиза и формы попадает в **одну Google Таблицу** — это база.
Никаких токенов и настроек, только вставить код и опубликовать.

```
Сайт (GitHub Pages)  ──POST──▶  Google Apps Script  ──▶  Google Таблица (база)
```

Таблица: <https://docs.google.com/spreadsheets/d/1M_Io_Yumspp6AH0zFGbkUFvdWaWFIT22oZ9bJUIMuhs/edit>

---

## Шаг 1. Привязать скрипт к таблице

1. Откройте вашу таблицу (ссылка выше).
2. **Расширения → Apps Script**.
3. Удалите содержимое `Code.gs` и вставьте код из [`apps-script/Code.gs`](apps-script/Code.gs)
   этого репозитория. Сохраните (Ctrl/Cmd+S).

---

## Шаг 1b. Построить аккуратный лист (один раз)

Скрипт делает из листа мини-CRM: ответы квиза по отдельным столбцам, закреплённая
синяя шапка, формат даты, столбец **Статус** с выпадающим списком и подсветкой,
столбец **Комментарий**, чередующиеся полосы.

1. В редакторе Apps Script вверху, в выпадающем списке функций, выберите **`setup`**.
2. Нажмите **Run** (▶). Если попросит разрешения — выдайте (как при первой настройке).
3. Готово: лист `Leads` пересоздан и красиво оформлен.

> ⚠️ `setup` **пересоздаёт** лист `Leads` (удаляет старые/тестовые строки). Запускайте,
> пока в нём нет реальных заявок. Потом не запускайте — иначе сотрёт данные.

Столбцы итогового листа:
`Дата · Источник · Имя · Телефон · Кто заполняет · Образование · Язык · Бюджет ·
Когда поступать · Статус · Комментарий · Источник трафика`

## Шаг 2. Опубликовать скрипт

**Первый раз:**
1. Apps Script: **Deploy → New deployment → ⚙ → Web app**.
2. **Execute as:** Me. **Who has access:** Anyone.
3. **Deploy** → разрешите доступ → скопируйте **Web app URL** (`.../exec`).
4. Проверка: откройте URL в браузере — покажет `{"ok":true,"status":"lead endpoint alive"}`.

**Если меняли код (как сейчас):** обновите деплой, чтобы сайт начал использовать новый код —
**Deploy → Manage deployments → ✎ (карандаш) → Version: New version → Deploy**.
Адрес `/exec` остаётся прежним, сайт менять не нужно.

---

## Шаг 3. Вставить адрес в сайт

Пришлите мне ваш `/exec` URL — **вставлю в код, закоммичу и запушу за вас**.

Либо сами: откройте [`src/lib/leads.ts`](src/lib/leads.ts), замените заглушку и запушьте:

```ts
const LEAD_ENDPOINT_FALLBACK = "https://script.google.com/macros/s/AKfyc.../exec";
```

```bash
git add -A && git commit -m "feat: подключить приём заявок" && git push
```

## Шаг 4. Проверка

1. Откройте сайт, пройдите квиз и оставьте телефон.
2. В Google Таблице появилась новая строка с заявкой.

Пока адрес не вставлен, сайт работает как обычно, а заявки пишутся в консоль браузера
(F12 → Console) — ничего не теряется во время настройки.

## (Позже) Хотите ещё и уведомления в Telegram?

Код уже готов — Telegram просто выключен. Чтобы включить: в Apps Script
**⚙ Project Settings → Script Properties** добавьте две строки —
`TELEGRAM_TOKEN` и `TELEGRAM_CHAT_ID` — и каждая новая заявка начнёт дублироваться
в Telegram. Менять код не нужно. Скажите — помогу с настройкой.
